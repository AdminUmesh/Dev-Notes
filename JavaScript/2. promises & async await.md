## Handling Async Flow using Promises and Async/Await

# JavaScript Promises

### What is a Promise?

A Promise in JavaScript is an object that represents the eventual
completion (or failure) of an asynchronous operation.

### Promise States:

-   Pending
-   Fulfilled
-   Rejected

------------------------------------------------------------------------


## Creating a Promise

``` js
const myPromise = new Promise((resolve, reject) => {
    let success = true;

    if (success) {
        resolve("Operation Successful");
    } else {
        reject("Operation Failed");
    }
});
```

------------------------------------------------------------------------

### Handle Using .then() and .catch()

``` js
myPromise
    .then(result => {
        console.log("Success:", result);
    })
    .catch(error => {
        console.log("Error:", error);
    });
```
### Handle Using async/await

```javaScript
async function handlePromise() {
    try {
        const result = await myPromise;
        console.log("Success:", result);
    } catch (error) {
        console.log("Error:", error);
    }
}

handlePromise();
```

# What is a Async/Await?
async/await is a way to handle asynchronous operations in JavaScript in a clean, readable way. `It is built on top of Promises.`

### 1. async

`Makes a function return a Promise`

```js
async function test() {
    return "Hello";
}
```

This actually returns:
```js
Promise.resolve("Hello")
```

### 2. await

`Waits for a Promise to resolve`
```js
const result = await somePromise;
```

✔ Pauses execution of that function
❌ Does NOT block the whole JavaScript thread

# Base Example

``` js
function createOrder() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            console.log("Order created");
            resolve("Order ID: 123");
        }, 1000);
    });
}

function preparation(orderId) {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            console.log("Food is being prepared");
            resolve("Food prepared");
        }, 2000);
    });
}

function foodReady(status) {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            console.log("Food is ready");
            resolve("Serve food");
        }, 1000);
    });
}
```

------------------------------------------------------------------------

## Using Promise Chaining (.then / .catch)

``` js
createOrder()
    .then(orderId => preparation(orderId))
    .then(status => foodReady(status))
    .then(result => console.log(result))
    .catch(err => console.log("Error:", err));
```

------------------------------------------------------------------------

## Using Async/Await

``` js
async function processOrder() {
    try {
        const orderId = await createOrder();
        const prep = await preparation(orderId);
        const result = await foodReady(prep);

        console.log(result);
    } catch (err) {
        console.log("Error:", err);
    }
}

processOrder();
```

------------------------------------------------------------------------

## Flow

    Order created → Food preparation → Food ready → Serve food

------------------------------------------------------------------------

## Key Points

-   Promises handle async operations
-   `.then()` is chaining
-   `.catch()` handles errors
-   `async/await` makes code cleaner
-   Both approaches work the same internally
