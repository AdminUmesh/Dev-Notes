## 🔹 What is AJAX?

AJAX = Asynchronous JavaScript and XML

-   Used to send/receive data from server **without reloading the page**
-   Makes web apps fast and dynamic
-   Works with:
    -   JSON (most common)
    -   XML
    -   HTML
    -   Text

------------------------------------------------------------------------

## 🔹 How AJAX Works

1.  User action (click, load, etc.)
2.  JavaScript sends request to server
3.  Server processes request
4.  Server sends response
5.  JavaScript updates UI (no reload)

------------------------------------------------------------------------

## 🔹 Example Explained

``` js
await $.ajax({
    url: "../Home/GetBillTax?GstName=" + GstName,
    type: "GET",
    datatype: 'json',
    success: function (TaxStruct) {
        console.log(TaxStruct);
    }
});
```

### Explanation:

-   `await` → waits for AJAX call to complete
-   `url` → API endpoint
-   `type` → HTTP method (GET, POST, etc.)
-   `datatype` → expected response type
-   `success` → runs when request succeeds

------------------------------------------------------------------------

## 🔹 Common AJAX Options

  Option     Description
  ---------- --------------------------
  url        API endpoint
  type       GET, POST, PUT, DELETE
  data       Data sent to server
  success    Success callback
  error      Error callback
  datatype   Expected response format

------------------------------------------------------------------------

## 🔹 GET Request Example

``` js
$.ajax({
    url: "/api/users",
    type: "GET",
    success: function(data) {
        console.log(data);
    }
});
```

------------------------------------------------------------------------

## 🔹 POST Request Example

``` js
$.ajax({
    url: "/api/save",
    type: "POST",
    data: { name: "Umesh" },
    success: function(res) {
        console.log(res);
    }
});
```

------------------------------------------------------------------------

## 🔹 Using AJAX with async/await

``` js
async function getData() {
    try {
        const result = await $.ajax({
            url: "/api/data",
            type: "GET"
        });

        console.log(result);
    } catch (err) {
        console.log(err);
    }
}
```

# 2. What is XMLHttpRequest (XHR)?

XMLHttpRequest is the **original way to make AJAX calls in JavaScript**
(before jQuery & fetch).

------------------------------------------------------------------------

## 🔹 XHR Example Explained

``` js
const xhr = new XMLHttpRequest();

xhr.open("GET", "https://api.example.com/data", true);

xhr.onreadystatechange = function () {
  if (xhr.readyState === 4 && xhr.status === 200) {
    console.log(xhr.responseText);
  }
};

xhr.send();
```

### Explanation:

-   `XMLHttpRequest()` → creates request object
-   `open()` → setup request (method, URL, async)
-   `onreadystatechange` → listens for response
-   `readyState === 4` → request complete
-   `status === 200` → success
-   `responseText` → data from server
-   `send()` → sends request

------------------------------------------------------------------------

## 🔹 XHR ReadyState Values

  Value   Meaning
  ------- -------------------------------
  0       Request not initialized
  1       Server connection established
  2       Request received
  3       Processing request
  4       Request finished

------------------------------------------------------------------------

## 🔹 Problems with XHR

-   Complex syntax
-   Hard to read
-   Callback-based

------------------------------------------------------------------------

## 🔹 Modern Alternative

-   jQuery AJAX (simpler)

## 🔹 Key Points

-   AJAX allows async server communication
-   No page reload required
-   jQuery simplifies AJAX
-   XHR is old but important concept
