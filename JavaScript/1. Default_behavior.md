# JavaScript Default Behavior (Synchronous vs Asynchronous)

`JavaScript is SINGLE-THREADED and SYNCHRONOUS by default` But it can behave asynchronously using the browser/Node environment

## 1. When JavaScript behaves SYNCHRONOUS (blocking)
If a function takes time and runs on the main thread → it will freeze everything

## Example
```javascript
function heavyTask() {
    let start = Date.now();
    while (Date.now() - start < 3000) {
        // blocking loop for 3 seconds
    }
    console.log("Done");
}


console.log("Start");
heavyTask();
console.log("End");
```

**Output:**
```base
Start
(3 sec pause)
Done
End
```

- ✔️ UI freezes
- ✔️ Nothing else runs
- ✔️ This is blocking (synchronous)

## 2. When JavaScript behaves ASYNCHRONOUS (non-blocking)

## Example
``` js
console.log("Start");

setTimeout(() => {
    console.log("Inside Timeout");
}, 2000);

console.log("End");
```

**Output**

    Start
    End
    (after 2 sec)
    Inside Timeout

------------------------------------------------------------------------

## Explanation

-   JavaScript runs code **synchronously (line by line)**.
-   `setTimeout` is handled by **Web APIs (browser/Node)**.
-   JS does NOT wait for `setTimeout`.
-   Callback goes to **callback queue**.
-   Event loop pushes it back after delay.

------------------------------------------------------------------------

## Key Points

-   JavaScript is **single-threaded**
-   Default behavior is **synchronous**
-   Async operations are handled by environment
-   `setTimeout` is **non-blocking**
