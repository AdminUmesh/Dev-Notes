## What is `fetch()`?

`fetch()` is a built-in JavaScript function used to make HTTP requests.`fetch() is an asynchronous function and it returns a Promise`

You use it to:
- get data from an API
- send data to a server
- update data
- delete data

Example:

```js
fetch('https://api.example.com/users')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error(error));
```

---

## Is `fetch()` built in, or do you need to install it?


### In browsers
`fetch()` is built in.
You do **not** need to install anything in modern browsers.

### In Node.js
- In modern Node.js versions, `fetch()` is also built in.
- In older Node.js versions, you had to install packages like `node-fetch`.

So in most current JavaScript environments, `fetch()` is already available.

---

## Syntax of `fetch()`

```js
fetch(url, options)
```

### Parameters

### 1. `url`
The API or resource address.

Example:

```js
fetch('https://api.example.com/products')
```

### 2. `options` (optional)
An object used to configure the request.

Common options:
- `method`
- `headers`
- `body`
- `mode`
- `credentials`
- `cache`
- `redirect`
- `signal`

Example:

```js
fetch('https://api.example.com/products', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ name: 'Phone', price: 20000 })
})
```

---

## Common HTTP methods used with `fetch()`

These are the main request methods:

### 1. `GET`
Used to fetch data.

```js
fetch('https://api.example.com/users')
```

### 2. `POST`
Used to send new data.

```js
fetch('https://api.example.com/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ name: 'Umesh' })
})
```

### 3. `PUT`
Used to fully update existing data.

```js
fetch('https://api.example.com/users/1', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ name: 'Umesh Kumar Singh' })
})
```

### 4. `PATCH`
Used to partially update data.

```js
fetch('https://api.example.com/users/1', {
  method: 'PATCH',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ name: 'Umesh' })
})
```

### 5. `DELETE`
Used to remove data.

```js
fetch('https://api.example.com/users/1', {
  method: 'DELETE'
})
```

## Important methods and properties used with fetch response

`fetch()` returns a **Promise**.

When the promise resolves, you get a `Response` object.

Common response methods:

### 1. `response.json()`
Convert response data to JavaScript object.

### 2. `response.text()`
Read response as plain text.

### 3. `response.blob()`
Read response as binary large object, often used for files/images.

### 4. `response.formData()`
Read response as form data.

### 5. `response.arrayBuffer()`
Read response as raw binary data.

Common response properties:
- `response.ok`
- `response.status`
- `response.statusText`
- `response.headers`
- `response.url`

Example:

```js
fetch('https://api.example.com/users')
  .then(response => {
    if (!response.ok) {
      throw new Error(`HTTP error: ${response.status}`);
    }
    return response.json();
  })
  .then(data => console.log(data))
  .catch(error => console.error(error));
```

## Using `async/await` with `fetch()`

```js
async function getUsers() {
  try {
    const response = await fetch('https://api.example.com/users');

    if (!response.ok) {
      throw new Error(`HTTP error: ${response.status}`);
    }

    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}
```

---

## How `fetch()` works step by step

1. Call `fetch(url, options)`
2. Request goes to the server
3. Server sends back a response
4. `fetch()` resolves with a `Response` object
5. Read the response using `.json()`, `.text()`, etc.
6. Handle errors if needed

---

## Does `fetch()` fail on HTTP error?

This is important:

`fetch()` does **not** automatically reject for HTTP errors like `404` or `500`.
It only rejects for network errors.

That is why you should check:

```js
if (!response.ok) {
  throw new Error('Request failed');
}
```

## Most common options in `fetch()`

### `method`
HTTP method like `GET`, `POST`, `PUT`, `PATCH`, `DELETE`

### `headers`
Used to send extra information, such as content type or authorization token

### `body`
Used to send data in request

### `signal`
Used to cancel request with `AbortController`

Example:

```js
const controller = new AbortController();

fetch('https://api.example.com/users', {
  signal: controller.signal
});

controller.abort();
```


## Practice example

```js
async function createUser() {
  try {
    const response = await fetch('https://jsonplaceholder.typicode.com/users', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        name: 'Umesh',
        email: 'umesh@example.com'
      })
    });

    if (!response.ok) {
      throw new Error('Failed to create user');
    }

    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}
```
