### **String**
Strings are the type of objects that can store the character of values.

### **string vs System.String**
`In C#, the string keyword is an alias for String; therefore, String and string are equivalent. It's recommended to use the provided alias string as it works even without using System;.`

### **Declaring and initializing strings**
```csharp
string message1; // Declare without initializing.
string? message2 = null; // Initialize to null.
string message3 = System.String.Empty; // Initialize as an empty string instead of the literal "".
System.String greeting = "Hello World!"; // Use System.String.
const string message4 = "You can't get rid of me!"; // Use a const string to prevent 'message4'
char[] letters = { 'A', 'B', 'C' }; // Use the String constructor only when creating a string from a char*, char[], or sbyte*.
string alphabet = new string(letters); 
```

### **There are two ways to create a string in C#:**

```csharp
// Without new Keyword (String Literal):
string str1 = "Hello, world!";
// With new Keyword (String Constructor):
string str2 = new string("Hello, world!");
```

### **Immutability of strings**
Strings in C# are immutable, meaning they cannot be changed after creation.

- Any modification creates a new string object
- Original string remains unchanged
- Old object is eligible for garbage collection

```csharp
string s1 = "A string is more ";
string s2 = "than the sum of its chars.";

s1 += s2; // creates new string
Console.WriteLine(s1);
```

### **String interpolation**
String interpolation is a feature in C# that allows you to insert variables and expressions directly into a string using $ and {}.

**Syntax**
```csharp
$"Text {variable}"
```

**Example**
```csharp
var jh = (firstName: "Jupiter", lastName: "Hammon", born: 1711, published: 1761);

Console.WriteLine($"{jh.firstName} {jh.lastName} was born in {jh.born}");
Console.WriteLine($"He was published at age {jh.published - jh.born}");
```

### Substrings
A substring is a part of a string (a sequence of characters taken from the original string).

```c#
string s3 = "Visual C# Express";
System.Console.WriteLine(s3.Substring(7, 2)); // Output: "C#"
System.Console.WriteLine(s3.Replace("C#", "Basic")); // Output: "Visual Basic Express"
int index = s3.IndexOf("C"); // index = 7 (Index values are zero-based)
```

### **Using StringBuilder for fast string creation**
StringBuilder is used when you need to perform multiple string modifications efficiently, especially inside loops.

**Why use StringBuilder?**
- Strings are immutable → every change creates a new object
- In loops or heavy operations → this can impact performance
- StringBuilder uses a mutable buffer → avoids creating new objects

**Key Benefits**
- Better performance for repeated string operations
- Can modify characters directly (unlike string)
- Efficient for large or dynamic string building

**Example:** Modify String Without Creating New Object
```csharp
var sb = new StringBuilder("Rat: the ideal pet");
sb[0] = 'C';
Console.WriteLine(sb.ToString());
// Output: Cat: the ideal pet
```

**Example:** Build String in Loop
```csharp
var sb = new StringBuilder();

for (int i = 0; i < 10; i++)
{
    sb.Append(i);
}
Console.WriteLine(sb); // 0123456789
```

**StringBuffer** vs **StringBuilder**
C# StringBuilder class is used to create mutable (modifiable) String. The C# StringBuilder class is same as StringBuffer class except that it is non-synchronized.

| **StringBuffer** | **StringBuilder** |
| ---------------- | ----------------- |
| heap area        | heap area |
| mutable object   | mutable object |
| consume less memory | consume less memory |
| all method are sychronised thus its thread safe. | non sychronised method nor thread safe. |
| fast as compare to string | fast as compare to stringBuffer |
| use when data is changing frequently. | use when data is changing frequently. |

**Why string objects are immutable in C#?**
Because C# uses the concept of string literal. Suppose there are 5 reference variables, all refer to one object “Sachin”. If one reference variable changes the value of the object, it will be affected by all the reference variables. That is why string objects are immutable in C#.

**Why C# uses the concept of string literal?**
To make C# more memory efficient (because no new objects are created if it exists already in the string constant pool).
